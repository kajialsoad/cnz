## কাঠামো
- Flutter মোবাইল/ডেস্কটপ অ্যাপ (রুট ডিরেক্টরি)
- Admin প্যানেল (Vite React) — `clean-care-admin`
- Backend API (Node + Prisma) — `server`

## রান অর্ডার
1) Backend API চালু (পোর্ট `4000`)  
2) Admin প্যানেল চালু (ডিফল্ট `5173`)  
3) Flutter অ্যাপ চালু (Windows ডেস্কটপ বা Web/Android)

## প্রস্তুতি
- Node 18+ ইন্সটল আছে নিশ্চিত
- Flutter SDK ও Android Studio/Windows Desktop enable (প্রয়োজনে `flutter doctor`)
- সার্ভার `.env` প্রস্তুত (পাওয়া গেছে) এবং ডাটাবেস কানেকশন সচল

## কমান্ড (Windows)
### 1) Backend
- `cd server`
- `npm install`
- Prisma প্রস্তুতি (যদি প্রথম রান):  
  - `npm run prisma:generate`  
  - `npm run db:setup` (বা অন্তত `prisma db push`)
- ডেভ সার্ভার: `npm run dev`  
- যাচাই: `http://localhost:4000` লগে শুনছে কিনা দেখব

### 2) Admin প্যানেল
- `cd clean-care-admin`
- `npm install`
- ডেভ সার্ভার: `npm run dev`  
- ব্রাউজার: `http://localhost:5173`

### 3) Flutter অ্যাপ
- রুটে: `flutter pub get`
- Windows ডেস্কটপ: `flutter run -d windows`  
  - বিকল্প: Web `flutter run -d chrome`  
  - Android এমুলেটর: `flutter emulators --launch <name>` → `flutter run -d emulator-5554`
- অ্যাপের `ApiConfig.baseUrl` ডেভ মোডে `http://localhost:4000` — ব্যাকএন্ড আগে চালু থাকা দরকার

## যাচাই
- Backend লগে “Server listening on port 4000”/সমজাতীয় বার্তা
- Admin: ব্রাউজারে UI লোড হয়
- Flutter: হোম/লগইন স্ক্রীন আসে; API কল সফল

## সম্ভাব্য ব্লকার
- Prisma মাইগ্রেশন/ডিবি ক্রেডেনশিয়াল — ব্যর্থ হলে `.env` ঠিক করে `prisma db push`
- Android SDK/ড্রাইভার — ডেস্কটপ মোডে শুরু করুন বাধা পেলে
- পোর্ট কনফ্লিক্ট — 4000/5173 বদলাতে হলে কনফিগ আপডেট

## আমি কী করব
- তিনটি সার্ভিস উপরের কমান্ডে চালু করব
- চলমান সার্ভারগুলোর URL শেয়ার করব
- যদি কোনো ধাপে ত্রুটি আসে, তাৎক্ষণিক ফিক্স করে রিরান করব
