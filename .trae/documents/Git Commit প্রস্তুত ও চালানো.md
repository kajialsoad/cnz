## উদ্দেশ্য

* বর্তমান পরিবর্তনগুলো পরিষ্কারভাবে স্টেজ করে অর্থপূর্ণ একটি কমিট তৈরি করা

* কনভেনশনাল কমিট ফরম্যাটে (বাংলা বর্ণনা সহ) মেসেজ ব্যবহার করা

* প্রয়োজন হলে রিমোটে পুশ করা

## প্রস্তুতি যাচাই

* `git status` চালিয়ে কোন কোন ফাইল পরিবর্তিত/নতুন/ডিলিট হয়েছে দেখব

* অনাকাঙ্ক্ষিত ডিরেক্টরি বাদ দেব: `build/`, `.dart_tool/`, `.idea/`, `.vscode/`

* কোড স্টাইল ঠিক আছে কিনা দেখতে: `dart format .` এবং `flutter analyze`

## স্টেজিং

* সব ট্র্যাকড পরিবর্তন স্টেজ: `git add -A`

* যদি কিছু ফাইল বাদ দিতে হয়, সিলেক্টিভ স্টেজিং: `git add <file_path>`

* `.gitignore` নিশ্চিত করবে অপ্রয়োজনীয় ফাইল স্টেজ না হয়

## কমিট মেসেজ (বাংলা + কনভেনশনাল কমিট)

* ফরম্যাট: `<type>: <সারাংশ>`

* উদাহরণ:

  * `feat: নতুন কমপ্লেইন্ট সফল পেজ যুক্ত`

  * `fix: কমপ্লেইন্ট সফল পেজে ন্যাভিগেশন বাগ ঠিক`

  * `refactor: UI কোড পরিষ্কার ও পুনর্গঠন`

* কমিট কমান্ড: `git commit -m "feat: নতুন কমপ্লেইন্ট সফল পেজ যুক্ত"`

## যাচাই (কমিটের আগে/পরে)

* স্ট্যাটিক এনালাইসিস: `flutter analyze`

* (থাকলে) টেস্ট: `flutter test`

* দ্রুত রান চেক: `flutter run` অথবা কম্পাইল চেক: `flutter build apk --debug`

## পুশ (ঐচ্ছিক)

* বর্তমান ব্রাঞ্চে পুশ: `git push origin <branch-name>`

* ব্রাঞ্চ না থাকলে প্রথমবার সেটআপ: `git push -u origin <branch-name>`

## নিরাপত্তা ও ব্যাকআউট

* ভুল হলে শেষ কমিট মেসেজ এডিট: `git commit --amend`

* স্টেজিং আনডু: `git restore --staged <file>`

* কমিট রিভার্ট: `git revert <commit-hash>`
