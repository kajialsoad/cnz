# প্রজেক্ট অ্যাডমিন (ওয়ার্ড ভিত্তিক) এবং সুপার অ্যাডমিন সাইট পরিকল্পনা

## ১. ভূমিকা (Introduction)
এই প্রজেক্টের মূল লক্ষ্য হলো একটি "ওয়ার্ড ভিত্তিক" অ্যাডমিন প্যানেল তৈরি করা, যেখানে সুপার অ্যাডমিন পুরো সিস্টেম নিয়ন্ত্রণ করবেন এবং প্রজেক্ট অ্যাডমিনরা শুধুমাত্র তাদের নির্দিষ্ট ওয়ার্ডের দায়িত্ব পালন করবেন।

## ২. ইউজার রোল এবং পারমিশন (User Roles & Permissions)

### ক. সুপার অ্যাডমিন (Super Admin)
*   **ক্ষমতা**: সম্পূর্ণ সিস্টেমের অ্যাক্সেস।
*   **ড্যাশবোর্ড**: পুরো সিটি কর্পোরেশনের রিপোর্ট (মোট অভিযোগ, সমাধান, পেন্ডিং)।
*   **অ্যাডমিন ম্যানেজমেন্ট**: নতুন "ওয়ার্ড অ্যাডমিন" তৈরি করা এবং তাদের নির্দিষ্ট ওয়ার্ড অ্যাসাইন করা।
*   **লোকেশন ম্যানেজমেন্ট**: সিটি কর্পোরেশন, জোন এবং ওয়ার্ড তৈরি বা এডিট করা।
*   **রিপোর্ট**: সব এলাকার বিস্তারিত রিপোর্ট দেখা।

### খ. প্রজেক্ট অ্যাডমিন / ওয়ার্ড অ্যাডমিন (Project Admin - Ward Based)
*   **ক্ষমতা**: শুধুমাত্র নিজের অ্যাসাইন করা ওয়ার্ডের অ্যাক্সেস।
*   **ড্যাশবোর্ড**: নির্দিষ্ট ওয়ার্ডের পরিসংখ্যান (আজকের অভিযোগ, সমাধানের হার)।
*   **অভিযোগ ম্যানেজমেন্ট**: শুধুমাত্র নিজের ওয়ার্ডের অভিযোগ দেখা, স্ট্যাটাস পরিবর্তন করা এবং কর্মীদের অ্যাসাইন করা।
*   **কর্মী ম্যানেজমেন্ট**: নিজের ওয়ার্ডের সার্ভিস প্রোভাইডারদের (Service Providers) মনিটর করা।

## ৩. টেকনিক্যাল ইমপ্লিমেন্টেশন (Technical Implementation)

### ব্যাকএন্ড (Backend - Node.js/Prisma)
1.  **ডেটাবেস স্কিমা**: `User` টেবিলে `role` (ADMIN/SUPER_ADMIN) এবং `ward` ফিল্ড ব্যবহার করা।
2.  **API সিকিউরিটি**: 
    *   API রিকোয়েস্টে চেক করা: ইউজার কি `SUPER_ADMIN`?
    *   যদি না হয়, তবে `ADMIN` এর ক্ষেত্রে `ward` ফিল্টার বাধ্যতামূলক করা।

### ফ্রন্টএন্ড (Frontend - React/Vite)
1.  **লগিন ও অথেন্টিকেশন**: 
    *   লগিনের পর ইউজারের রোল এবং ওয়ার্ড লোকাল স্টোরেজে বা কন্টেক্সটে সংরক্ষণ করা।
2.  **অ্যাডমিন তৈরি পেজ (Super Admin Only)**: 
    *   নতুন অ্যাডমিন তৈরির সময় "Role" সিলেক্ট করার অপশন।
    *   যদি "Project Admin" সিলেক্ট করা হয়, তবে "Ward" ড্রপডাউন দেখানো হবে।
3.  **ডেটা ফিল্টারিং**:
    *   API কল করার সময় অটোমেটিকলি ওয়ার্ড প্যারামিটার পাঠানো (যদি ওয়ার্ড অ্যাডমিন হয়)।
    *   UI তে ফিল্টার অপশন ডিজেবল করে রাখা (ওয়ার্ড অ্যাডমিনের জন্য)।

## ৪. কাজের ধাপ (Action Plan)

### ধাপ ১: পরিকল্পনা ও সেটআপ
*   বর্তমান কোডবেস বিশ্লেষণ (হয়ে গেছে)।
*   রোল ম্যানেজমেন্ট লজিক ঠিক করা।

### ধাপ ২: ব্যাকএন্ড ডেভেলপমেন্ট
*   `createUser` API তে অ্যাডমিন তৈরির লজিক আপডেট করা।
*   `getComplaints` API তে রোল ভিত্তিক ফিল্টারিং নিশ্চিত করা।

### ধাপ ৩: ফ্রন্টএন্ড ডেভেলপমেন্ট
*   **User Management Page**: অ্যাডমিন তৈরির ফর্মে ওয়ার্ড সিলেকশন যোগ করা।
*   **Dashboard**: লগিন করা অ্যাডমিনের ওয়ার্ড অনুযায়ী ডেটা দেখানো।
*   **Sidebar/Navigation**: রোল অনুযায়ী মেনু আইটেম দেখানো বা লুকানো।

### ধাপ ৪: টেস্টিং (Testing)
*   সুপার অ্যাডমিন হিসেবে লগিন করে নতুন ওয়ার্ড অ্যাডমিন তৈরি করা।
*   ওয়ার্ড অ্যাডমিন হিসেবে লগিন করে নিশ্চিত করা যে অন্য ওয়ার্ডের ডেটা দেখা যাচ্ছে না।
