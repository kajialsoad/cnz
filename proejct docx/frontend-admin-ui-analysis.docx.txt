Clean Care Admin — Frontend UI Full Analysis

Executive Summary
- Admin UI React + Vite + TypeScript + MUI ভিত্তিক।
- Routing `App.tsx`-এ হার্ডকোডেড; `routeConfig.ts` কনফিগ আছে কিন্তু ব্যবহৃত নয়।
- Dashboard, Complaints, Users, Reports, Notifications, Settings পেজ বিদ্যমান; Admin Management রুট নেই/অসম্পূর্ণ ছিল।
- অনেক জায়গায় বাস্তব API ইন্টিগ্রেশন অনুপস্থিত; স্যাম্পল/প্লেসহোল্ডার ডাটা ব্যবহার করা হয়েছে।

Architecture Overview
- Routing: `clean-care-admin/src/App.tsx` — `ProtectedRoute` দিয়ে গার্ডেড পেজ।
- Layout: `MainLayout` (Header + Sidebar + Content) — Sidebar menu `Sidebar.tsx:27`-এ ডিফাইনড।
- State/Context: `AuthContext.tsx` অথ স্টেট; `LanguageContext.tsx` i18n টগল।
- Services: `services/*` — `authService.ts`, `userManagementService.ts`, `translationService.ts`।
- Types: `types/*` — API/ইউজার/কমপ্লেইন্ট/report ইত্যাদির টাইপ ডিফিনিশন।
- Config: `apiConfig.ts`, `routeConfig.ts`, `environmentConfig.ts`, `appConfig.ts`।

Routing Map
- Public: `/login` → `Login.tsx` (`clean-care-admin/src/pages/Login/Login.tsx`).
- Protected:
  - `/` → `Dashboard.tsx` (`clean-care-admin/src/pages/Dashboard/Dashboard.tsx`).
  - `/complaints` → `AllComplaints.tsx` (`clean-care-admin/src/pages/AllComplaints/AllComplaints.tsx`).
  - `/users` → `UserManagement.tsx` (`clean-care-admin/src/pages/UserManagement/UserManagement.tsx`).
  - `/reports` → `Reports.tsx` (`clean-care-admin/src/pages/Reports/Reports.tsx`).
  - `/notifications` → `Notifications.tsx` (`clean-care-admin/src/pages/Notifications/Notifications.tsx`).
  - `/settings` → `Settings.tsx` (`clean-care-admin/src/pages/Settings/Settings.tsx`).
- Config-only (ম্যাপড নয়): `/admins`, `/admins/roles`, `/admins/permissions` (`routeConfig.ts:17–23`).

Layout & Navigation
- MainLayout: `components/common/Layout/MainLayout/MainLayout.tsx` — পেজ টাইটেল ও কন্টেন্ট container।
- Header: `Header.tsx` — সার্চ, ল্যাং সিলেক্টর, নোটিফিকেশন ব্যাজ, ইউজার অ্যাভাটার।
- Sidebar: `Sidebar.tsx:27–72` — মেনু আইটেম; `'/admins'` আইটেম আছে কিন্তু রুট ছিল না।
- Footer note: Sidebar bottom-এ ব্র্যান্ডিং (Smart Complaint System, 2025)।

Dashboard Composition
- StatsCards: `pages/Dashboard/components/StatsCards/*` — সামারি কার্ড।
- MiddleDashboardWidgets: `pages/Dashboard/components/MiddleDashboardWidgets/*` — চ্যানেল/চার্ট ব্লক।
- ComplaintsChart: `ComplaintStatusOverview.tsx`, `CitizenSatisfactionScore.tsx`, `AverageServiceTime.tsx`।
- BottomDashboardSection: `WardZonePerformance.tsx`, `TotalUsersWidget.tsx`, `WeeklyTrendAnalysis.tsx`, `OperationalMonitoring.tsx`।

User Management
- Page: `UserManagement.tsx` — লিস্ট, সার্চ/ফিল্টার, CRUD মডাল, স্ট্যাটস।
- Modals: `components/UserManagement/*` — Add/Edit/Details।
- Service: `userManagementService.ts` — `/api/admin/users` এন্ডপয়েন্টে কল।
- UX: Debounce সার্চ, টোস্ট নোটিফিকেশন, লোডিং স্কেলেটন, এম্পটি স্টেট।

API Integration Status
- `API_CONFIG`: `apiConfig.ts:1–55` — Admin auth, users, complaints, reports, notifications, settings এন্ডপয়েন্ট কনফিগ।
- বাস্তবে ব্যবহৃত:
  - Auth: `authService.ts` (admin login/profile)।
  - Users: `userManagementService.ts` (list/get/update/create)।
- ব্যবহৃত নয়/অনুপস্থিত UI:
  - Admins: `/admins`, `/admins/roles`, `/admins/permissions` UI ও রাউট অনুপস্থিত ছিল।
  - Reports/Notifications/Settings — অনেক ফিচার প্লেসহোল্ডার।

Internationalization (i18n)
- `LanguageContext.tsx` — ল্যাং টগল সাপোর্ট; `TranslatedText.tsx` কমন উইজেট।
- বর্তমান কন্টেন্টের উল্লেখযোগ্য অংশ ইংরেজিতে; বাংলা কনটেন্ট প্রয়োজন যেখানে ইউজার-ফেসিং।

Accessibility & UX
- Keyboard focus, aria-labels MUI ডিফল্টে ভালো, তবে:
  - ট্যাব-অর্ডার রিভিউ দরকার (সার্চ/ফিল্টার)।
  - রঙ কনট্রাস্ট চেক (গ্রেডিয়েন্ট বাটন/কার্ড)।
  - স্ক্রিনরিডার টেক্সট কিছু জায়গায় যোগ করা দরকার।

Performance
- Vite + MUI পারফরম্যান্স ঠিক আছে।
- Charts/Widgets lazy-load করলে initial paint হালকা হবে।
- সার্চ debounce আছে; টেবিল ভার্চুয়ালাইজেশন large dataset-এ বিবেচ্য।

Security
- `ProtectedRoute.tsx:32–35` — অথ গার্ড; অনঅথenticated হলে `/login`।
- Admin RBAC UI স্তরে নেই; ব্যাকএন্ডে RBAC আছে। UI-তে রোল-ভিত্তিক ভিজিবিলিটি দরকার।

Gaps / Missing Pieces
- Admin Management UI ও রাউট (admins/roles/permissions) অনুপস্থিত।
- Reports/Notifications/Settings সাবপেজ (templates/history/general/security/system/integrations) বাস্তবায়ন হয়নি।
- Complaints পেজে রিয়েল API ইন্টিগ্রেশন নেই; স্যাম্পল data।
- Navigation duplication: `components/layout/AdminNavbar.tsx` বনাম `Header+Sidebar` — কনসোলিডেশন দরকার।

Recommended Tasks (Frontend)
- Add Admin Management pages: Admins, Roles, Permissions (routes + placeholder UI)।
- Wire `routeConfig.ts` with router; হার্ডকোডেড রুট বাদ দিয়ে কনফিগ-ড্রিভেন রাউটিং।
- Integrate real API in Complaints/Reports/Notifications/Settings।
- Implement RBAC UI (হাইড/ডিসেবল অ্যাকশন by role)।
- i18n coverage বৃদ্ধি; বাংলা কনটেন্ট/লেবেল সম্পূর্ণ করা।
- Lazy-load চার্ট/উইজেট; টেবিল ভার্চুয়ালাইজেশন।
- Unify navigation (একটি লেআউট): `MainLayout` রিটেইন, `AdminNavbar` অপসারণ/আর্কাইভ।
